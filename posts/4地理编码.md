地理编码是将 SAR 雷达坐标系的数据转为地理坐标系的步骤，这涉及到两个概念：

- **SAR 雷达坐标系**：以雷达斜距向和方位向为基准的坐标系
- **地理坐标系**：以经纬度为基准的坐标系

## 0. 导入 DEM

如果 DEM 是自己裁剪的范围(提前在arcgis中镶嵌对应范围的dem)，一般是 TIFF 格式，需要在 GAMMA 中使用 :cmd[dem_import] 指令。

> **注意**：大地水准面用于将 DEM 高度从海拔高（相对于大地水准面）转换为椭球高。GAMMA 自带 EGM96 和 EGM2008 全球水准面文件，位于 `$DIFF_HOME/scripts/` 目录下。

## 1. 生成多视强度图

对 SLC 影像进行多视处理，生成多视强度图（MLI），提高信噪比并使距离向和方位向的像素间距接近一致。使用 :cmd[multi_look] 指令。

为了使 DEM 和雷达影像之间生成查找表，需要对 DEM 进行过采样，使其像素间距与雷达影像的像素间距大致相等。

1. 提取雷达影像的空间分辨率与入射角：

   ```bash
   r_space=`awk '$1 == “range_pixel_spacing:” {print $2}' $ref_slc.mli.par`
   az_space=`awk '$1 == “azimuth_pixel_spacing:” {print $2}' $ref_slc.mli.par`
   inc_angle=`awk '$1 == “incidence_angle:” {print $2}' $ref_slc.mli.par`
   ```

2. 将斜距转换为地距：

   ```bash
   ground_r_space=$(echo “scale=3;$r_space/s($inc_angle\*3.1415926/180)”|bc -l)
   ```

3. 将 DEM 的经纬度分辨率转换为”米”（不同地区纬度长度不同，本次实验为甘肃，长度大约为 90000 m，在赤道大约为 108000 m）：

   ```bash
   degree2meter=”90000”
   res_lat_m=`awk '$1 == “post_lat:” {print sqrt($2*$2)*'$degree2meter'}' $DEM_input_par`
   res_lon_m=`awk '$1 == “post_lon:” {print sqrt($2*$2)*'$degree2meter'}' $DEM_input_par`
   ```

4. 计算最终的过采样因子：

   ```bash
   lat_ovr=`printf “%.0f\n” $(echo “scale=3;$res_lat_m/$az_space”|bc)`
   lon_ovr=`printf “%.0f\n” $(echo “scale=3;$res_lon_m/$ground_r_space”|bc)`
   ```

## 2. 粗查找表与模拟 SAR 生成

根据过采样因子，结合精确轨道数据和DEM，生成初始的粗查找表，并将 DEM 转换生成雷达坐标系下的模拟 SAR 强度图。

流程：:cmd[gc_map] → :cmd[geocode]
:cmd[gc_map]： 结合雷达参数与外部 DEM，计算出初始坐标查找表，并根据地形起伏生成模拟 SAR 强度影像。
:cmd[geocode]：依靠初始查找表，将地图坐标系的模拟 SAR 影像转换至雷达几何坐标系。

## 3. 精配准

创建偏移量参数文件，首先进行全局宏观偏移的初始估计。随后对真实多视强度图（MLI）与模拟 SAR 图像执行多视互相关匹配，并剔除误差点，计算出两者之间的精确映射关系。

流程：:cmd[create_diff_par] → :cmd[init_offsetm] → :cmd[offset_pwr_trackingm2] → :cmd[offset_fitm]

## 4. 生成精查找表与坐标系转换

将上一步拟合好的偏移量参数应用到粗查找表中，生成用于最终转换的精细查找表。利用该表，将地理坐标系下的 DEM 精确重采样到雷达坐标系下。

流程：:cmd[gc_map_fine] → :cmd[geocode] → :cmd[rasdt_pwr]
